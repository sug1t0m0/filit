# bundle URL 仕様 (ステートレス化)

> **Status:** 確定 (2026-07-09 レビュー済み)
> **Last updated:** 2026-07-09
> 実装安定後に design.md へ統合する。

## 1. 目的

- `/bundle/<payload>` の **URL 単体から表示コンテンツを完全に復元**できるようにする (ステートレス)。
- サーバ側の bundle 登録 (`.filit/bundles.json`) を廃止する。URL の解釈に必要な状態をサーバが持たない。
- git のコミットハッシュを使い、**選択だけでなく内容まで** URL で固定できるモードを提供する。
- エンコード / デコードが対で定義され、どちらの方向にも機械的に変換できる。

ステートレスの対象外 (残る状態):

- `.filit/comments.json` — ユーザが書いた一次データ。URL から導出できる性質のものではない。
- git リポジトリ自体 — `rev` の解決に使う。

## 2. パラメータ (v1)

| キー | 型         | 必須 | 意味                                                                                                                                                       |
| ---- | ---------- | ---- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `f`  | `string[]` | ✓    | リポジトリ相対パスの**順序付き**リスト。順序は出力順 (R13 を先取り)                                                                                        |
| `r`  | `string`   | -    | git commit SHA (完全形 or 7桁以上の短縮形)。**指定時はそのコミット時点の内容** (`git show <r>:<path>`) を表示。省略時はワーキングツリーの現在内容 (ライブ) |

将来の拡張候補 (v1 では実装しない。キーだけ予約):

| キー | 意味                                        | 関連要件 |
| ---- | ------------------------------------------- | -------- |
| `l`  | ファイルごとの行範囲指定 (部分含有)         | -        |
| `p`  | 分割タブの部数情報 `{index, total}`         | R12      |
| `i`  | 冒頭に差し込む指示テキスト (プロンプト前文) | R14      |
| `t`  | bundle の表示タイトル                       | R11      |

## 3. エンコーディング

```
payload := <version文字><body>
version "1": body = base64url( deflate-raw( UTF-8( canonical-JSON(params) ) ) )
```

- **canonical-JSON**: キーを `f` → `r` の順で出力。同じパラメータからは常に同じ JSON 文字列が得られる (URL の文字列一致をおおむね保証するため。厳密な同一性判定はデコード結果で行う)。
- **deflate-raw**: 共通プレフィックスの多いパスリストが大幅に縮む。実装は `fflate` (同期 API・ブラウザ/Node 両対応・~8KB) を想定。
- **base64url**: `A-Z a-z 0-9 - _`。パスセグメントに安全に入り、既存ルート `/bundle/:payload` をそのまま使える。
- 先頭のバージョン文字は**圧縮の外**に置く。将来エンコーディング自体を変えられる (`2` = 別方式など)。

### 例

```json
{ "f": ["src/client/App.tsx", "src/server/server.ts"], "r": "9c1f2ab" }
```

→ `/bundle/1eJyrVkrLLNZLzs8rSc0r0S9OTsvMSVWyUjLUM0jJT...` (イメージ)

### サイズ見積もり

| ファイル数 | パス合計 (生) | payload 目安 |
| ---------- | ------------- | ------------ |
| 5          | ~150 B        | ~150 字      |
| 50         | ~1.5 KB       | ~600–800 字  |
| 300        | ~9 KB         | ~3–4K 字     |

- エンコーダは payload が **2,000 字超で警告**、**8,000 字超でエラー** (ブラウザ/HTTP 実装の相互運用限界の手前)。
- 上限に当たるケースの逃げ道は将来の R11 (名前付き選択セット = 長い URL の短い別名) で用意する。

## 4. エンコード / デコードの場所

- **エンコードはクライアント (ブラウザ) で完結**する。`POST /api/bundles` は廃止。「タブに開く」は選択リストから URL を組み立てて `window.open` するだけになる (サーバ往復ゼロ)。
- デコードはサーバ (`GET /api/bundles/:payload` / `:payload/raw`) とクライアント (表示用) の両方で行う。実装は `src/utils/bundleCode.ts` に isomorphic に置き、encode/decode のラウンドトリップをユニットテストで保証する。

### API (変更後)

| メソッド/パス                   | 説明                                                                            |
| ------------------------------- | ------------------------------------------------------------------------------- |
| `GET /api/bundles/:payload`     | payload をデコード → 解決した files + comments + bytes + missing を JSON で返す |
| `GET /api/bundles/:payload/raw` | 同上、素の合成 Markdown                                                         |
| `GET /bundle/:payload`          | SPA ルート (リッチページ)                                                       |

`POST /api/bundles` は削除。`.filit/bundles.json` と `BundleStore` は削除。

## 5. `r` (rev) の解決とセキュリティ

- `r` 指定時: `git show <r>:<path>` で読む。リポジトリが無い / rev が存在しない / そのコミットにファイルが無い場合は該当ファイルを missing 扱い (ページに警告表示)。
- `r` 省略時: 現行どおりワーキングツリーから読む。
- **スコープ検査は rev モードでも適用する**: `f` のパスが CLI 起動時のスコープ外なら拒否 (URL は外から与えられるため、デコード後のパスは信頼しない)。
- デコード安全策: 展開後 JSON は 256 KB 上限 (zip bomb 対策)、スキーマ検査 (`f` は string[]、パスに `..` / 絶対パス禁止、`r` は `[0-9a-f]{7,40}`)。

## 6. コメントの扱い

- v1 では **rev に関係なく `.filit/comments.json` をライブ解決**して重ねる (現行踏襲)。
- `r` 固定の bundle では、コメント作成時点とファイル内容が異なり行アンカーがズレる可能性がある → NF6 (ベストエフォート) の範囲とし、将来の再アンカリング (R15) で改善する。

## 7. UI (決定)

- ヘッダに**2つのボタンを並べて提供**する (Q1 決定: 両方選べる):
  - **タブに開く** — rev なし (ライブ)
  - **HEAD 固定で開く** — `r` = HEAD の SHA。git リポジトリでない場合は非表示
- HEAD の SHA と dirty 状態は `GET /api/git/head` (`{ sha, dirty: string[] }`) で取得する。
- 選択に未コミット変更 (dirty) のファイルが含まれる状態で「HEAD 固定」を押した場合は、**対象ファイルを列挙した確認ダイアログで警告して続行** (Q2 決定)。固定ページのヘッダに `@9c1f2ab` のようにコミットを表示する。
- payload が 2,000 字を超える場合は確認ダイアログで警告、8,000 字超はエラー。
